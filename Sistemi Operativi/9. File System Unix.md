_**FILE SYSTEM DI UNIX**_
ad ogni file possono essere associati uno o più nomi simbolici ma un solo descrittore **(i-node)** univocamnete identificato da un intero **(i-number)** 

metodo di allocazione **a indice** e formattazione del disco in blocchi
la [[partizione del File System]] è:

![[Screenshot 2026-03-18 105409.png|174]]

**INDIRIZZAMENTO**
precisamente **a indice con più livelli di indirizzamento**

l'allocazione non avviene su dei blocchi fisicamente contigui, infatti nell' i-node sono contenuti dei **puntatori a blocchi** 

![[Screenshot 2026-03-18 110136.png|373]]

10 blocchi (di dato) sono acessibili direttamente: 5 KB
128 blocchi di dato (puntatore 11) sono accessibili con indirezione singola: 64 Kb
128 * 128 blocchi di dato (puntatore 12) sono accessibili con indirezione doppia: 8MB
128 * 128 * 128 blocchi di dato (puntatore 13) sono accessibili con indirezione tripla: 1GB

_dimensione massima_ = 1GB + 8MB + 64KB + 5KB

**PROTEZIONE**
- 3 modalità di a ccesso: lettura, scrittura, 
- il proprietario può concedere o negare agli altri utenti il permesso di accedere ai propri file
- utente privilegiato _root_ ha accesso a tutti i file 

ogni file ha **12 bit di protezione** 
![[Screenshot 2026-03-18 110840.png|400]]

di cui **9 per lettura (r), scrittura (w) ed esecuzione (x)** 
U -> user
G -> group
O -> others
ad esempio
![[Screenshot 2026-03-18 111009.png|375]]

e **3 bit di permessi per gli eseguibili** 
_SUID_: set user id, se settato associa al processo che esegue il file lo User ID del proprietario del file
_SGID_: set group id, se settato associa al processo che esegue il file il Group ID del proprietario del file
_Sticky_: save text image, l'immagine del processo viene mantenuta in area di swap  anche dopo che ah finito il proprio compito

**ACCESSO AL FILE SYSTEM**
concetti generali:
- accesso sequenziale
- puntatore al file **I/O pointer** registra la posizione corrente
- assenza di strutturazione 
![[Screenshot 2026-03-18 112729.png|425]]

_file descriptor_
ad ogni processo è associata una **tabella dei file aperti**, ogni elemento tella tabella rappresenta un file aperto, individuato da un codice intero (file descriptor, 0,1,2 individuano standard input, output ed error), e questa tabella è allocata nella [[User Structure]] 
![[Screenshot 2026-03-18 113135.png|153]]

per realizzare l'accesso ai file il SO utilizza **due strutture dati globali**
- _tabella dei file attivi_: per ogni file aperto contiene una copia dell' i-node
- _tabella dei file aperti di sistema_: ha un elemento per ogni operazione di apertira relativa a file aperti ( I/O pointer e un puntatore all' i-node )

in merito alla condivisione, il figlio può accedere al file anche se l'apertura è avvenuta prima della fork in quanto eredita la User Structure, accedendo al file tramite o stesso I/O pointer,  ogni accesso al filedel figlio lo modificherà anche per il padre, e viceversa

## ==SYSTEM CALL DI ACCESSO AI FILE== 


## _**apertura di un file: OPEN **_ 
```c
int open(char nomefile[],int flag, [int mode]);
```
- _nomefile[]_: nome del file (relativo o assoluto)
- _flag_: esprime il modo di accesso (O_RDONLY, O_WRONLY, O_APPEND, O_CREAT accesso in scrittura e crea il file se non esiste, O_TRUNC accesso in scrittura troncando la lunghezza del file a 0)
- _mode_: richiesto solo se l'apertura determina la creazione del file (flag O_CREAT) e in tal caso specifica i bit di protezione 

restituisce **il valore del file descrtiptor al file o -1 in caso di errore**  


## _**creazione di un file: CREAT**_  
```c
int creat(char nomefile[], int mode);
```
- _nomefile_: nome file
- _mode_: specifica i 12 bit di protezione del nuovo file 

restituisce **il file descriptor del file o -1 se fa errore** 
[[esempi di open e create]]


## _**chiusura di un file: CLOSE**_ 
```c
int close(int fd);
```
- _fd_: descriptor del file da chiudere

restituisce **0 in caso di successo <0 in caso di insuccesso** 



## _**lettura di un file: READ**_ 
```c
int read(int fd, char * buf, int n);
```
- _fd_: descrittore del file
- _buf_: area in cui trasferire i byte letti
- _n_: numero caratteri da leggere 

restituisce **il numero di caratteri letti** in caso di successo 
(carattere End-Of-FIle che marca la fine del file)

N.B. l' I/O pointer viene spostato in avanti del numero dei caratteri letti 


## _**scrittura di un file: WRITE**_ 
```c
int write(int fd, char * buf, int n); 
```

- _fd_: descrittore del file
- _buf_: area cda cui trasferire i byte scritti
- _n_: numero di caratteri da scrivere

restituisce **il numero dei caratteri letti** in caso di successo 

[[esempi di read e write]] 


## _**accesso diretto: LSEEK**_ (spostare l' I/O pointer)
```c
lseek(int fd, int offset, int origine)
```

- _fd_: descrittore del file
- _offset_: spostamento in byte
- _origine_ che può valere: 
		0 se inizio file (SEEK_SET)
		1 posizione corrente (SEEk_CUR)
		2 fine file (SEEK_END) 

restituisce **un int chge rappresenta la nuova posizione** in caso di successo 
[[esempio lseek]]


## _**UNLINK**_ (cancellare o decrementare il numero di link)
```c
int unlink(char * name);
```
- _name_: nome del file

ritorna **0 se ok, altrimenti -1** 
generalmente l'effetto è quello di decrementare di 1 il numero dei link (nell'i-node) eliminando il nome specificato dal File System, se il numero di link è 0 allora viene cancellato il file 


## _**LINK**_ (aggiungere un link ad un file esistente)
```c
int link (char * oldname, char * newname) 
```
- _oldname_: nome del file esistente
- _newname_: nome associato al nuovo link


## ==SYSTEM CALL PER GESTIONE DI FILE E DIRECTORY== 


## _**ACCESS**_ (verificare i diriti di accesso di un utente)
```c
int access (char * pathname, int amode); 
```
- _pathname_: nome del file
- _amode_ esprime il diritto da verifiare: 
		00 -> existence
		01 -> execute access
		02 -> write access
		04 -> read access 

restituisce **0 in caso di successo, altrimenti -1** 



## _**STAT**_ (leggere attributi di un file)
```c
int stat(const char *path, struct stat *buf);
```
- _path_: nome del file
- _buf_: puntatore ad una [[struttura di tipo stat]] 

[[esempio di stat]]

## _**CHMOD**_ (protezione)
```c
int cmod(char * pathname, char * newmode);
```
- _pathname_: nome del file
- _newmode_: contiene i nuovi diritti

## _**CHOWN**_ 
per cambiare il proprietario e il gruppo di un file
```c
int chown(char * pathname, int owner, int group);
```
- _pathname_: nome del file
- _owner_: user id del nuovo proprietario 
- _group_: group id del guppo

N.B. puo essere eseguita solo dall'utente root 


## ==SYSTEM CALL PER I DIRETTORI==  
## _**CHDIR**_ (cambiare direttorio)
```c
int chdir(char * nomedir);
```
- _nomedir_: nome direttorio in cui entrare

restituisce **0 in caso di successo e -1 in caso di fallimento** 

## _**OPENDIR**_ (aprire un direttorio) 
```c
DIR * opendir(char * nomedir);
```
- _nomedir_: nome direttorio da aprire

restituisce un **puntatore a DIR** (dir consente di  riferirsi ad un direttorio aperto) 

## _**CLOSEDIR**_ (chiudere un direttorio)
```c
int closedir(DIR * dir);
```
- _dir_: puntatore alla directory da chiudere

restituisce **0 in caso di successo e -1 in caso di fallimento**



**--- elemento del direttorio DIRENT ---** 
rappresenta i file come record in un direttorio 
```c
struct dirent {
    long d_ino; /* i-number */
    off_t d_off; /* offset del prossimo */
    unsigned short d_reclen; /* lunghezza del record */
    unsigned short d_namelen; /* lunghezza del nome */
    char *d_name; /* nome del file */
}
```

## _**READDIR**_ (lettura di un direttorio)
```c
struct dirent * descr //descrittore del directory 
descr = readdir (DIR * dir)
```
- _dir_: puntatore alla directory da leggere (risultato opendir)


## _**MKDIR**_ (creazione di un direttorio)
```c
int mkdir(char * pathname, int mode);
```
- _pathname_: nome del direttorio DA CREARE
- _mode_: esprime i bit di protezione 

restituisce **0 in caso di successo, altrimenti <= 0**

_**COMUNICAZIONE TRA PROCESSI UNIX**_
i processo non condividono memoria (modello ad ambiente locale)

l'interazione può avvenire:
- con la **condivisione di file**
- con **strumenti di Inter Process Communication** 
		- [[pipe]] tra processi della stessa gerarchia
		- _fifo_ qualunque insieme di processi 
		- _socket_ 


## ==PIPE== 
##  _**PIPE**_ (creazione)
```c
int pipe(int fd[2]);
```
- _fd_: puntatore ad un vettore di 2 file descriptor 
		- fd[0] lato di lettura della pipe 
		- fd[1] lato di scrittura 

restituisce **0 se ha successo, <0 in caso di fallimento** 

![[Screenshot 2026-03-20 171511.png|368]]

**omogeneità con i file** 
si può accedere alla pipe mediante le system call: _read_ e _write_ 

![[Screenshot 2026-03-20 171648.png|375]]

se la pipe è vuota il processo che legge si blocca
se la pipe è piena il processo che scrive si blocca 

**processi che possono comunicare mediante la pipe**: 
- processi fratelli 
- processo padre e figlio 
- nonno e nipote
- etc.

**chiusura di pipe**
ogni processo può chiudere un estremo con una _close_

duplica un elemento nella tabella dei file aperti di processo
```c
int dup(int fd);
```
- _fd_: file descriptor del file da duplicare

restituisce **il nuovo file descriptor (file copiato) o -1 in caso di errore**

## ==FIFO== 
## _**MKFIFO**_ (creazione)
```c
int mkfifo(char* pathname, int mode);
```
- _pathname_: nome della fifo
- _mode_: esprime i permessi

restituisce **0 in caso di successo o <0 in caso di errore** 

**apertura/chiusura di una fifo** 
```c
int fd;
fd=open(“myfifo”, O_RDONLY);

//chiudere
close(fd);

//elminare
unlink("myfifo");
```

**accesso a fifo**
mediante _read_ e _write_
```c
int fd; 
char msg[10]; 
fd=open(“myfifo”, O_RDONLY); 
read(fd, msg,10);
```
