**INDIRIZZAMENTO**
precisamente **a indice con più livelli di indirizzamento**

l'allocazione non avviene su dei blocchi fisicamente contigui, infatti nell' i-node sono contenuti dei **puntatori a blocchi** 

![[Screenshot 2026-03-18 110136.png|373]]

10 blocchi (di dato) sono acessibili direttamente: 5 KB
128 blocchi di dato (puntatore 11) sono accessibili con indirezione singola: 64 Kb
128 * 128 blocchi di dato (puntatore 12) sono accessibili con indirezione doppia: 8MB
128 * 128 * 128 blocchi di dato (puntatore 13) sono accessibili con indirezione tripla: 1GB

_dimensione massima_ = 1GB + 8MB + 64KB + 5KB

**PROTEZIONE**
- 3 modalità di a ccesso: lettura, scrittura, 
- il proprietario può concedere o negare agli altri utenti il permesso di accedere ai propri file
- utente privilegiato _root_ ha accesso a tutti i file 

ogni file ha **12 bit di protezione** 
![[Screenshot 2026-03-18 110840.png|400]]

di cui **9 per lettura (r), scrittura (w) ed esecuzione (x)** 
U -> user
G -> group
O -> others
ad esempio
![[Screenshot 2026-03-18 111009.png|375]]

e **3 bit di permessi per gli eseguibili** 
_SUID_: set user id, se settato associa al processo che esegue il file lo User ID del proprietario del file
_SGID_: set group id, se settato associa al processo che esegue il file il Group ID del proprietario del file
_Sticky_: save text image, l'immagine del processo viene mantenuta in area di swap  anche dopo che ah finito il proprio compito

**ACCESSO AL FILE SYSTEM**
concetti generali:
- accesso sequenziale
- puntatore al file **I/O pointer** registra la posizione corrente
- assenza di strutturazione 
![[Screenshot 2026-03-18 112729.png|425]]

_file descriptor_
ad ogni processo è associata una **tabella dei file aperti**, ogni elemento tella tabella rappresenta un file aperto, individuato da un codice intero (file descriptor, 0,1,2 individuano standard input, output ed error), e questa tabella è allocata nella [[User Structure]] 
![[Screenshot 2026-03-18 113135.png|153]]

per realizzare l'accesso ai file il SO utilizza **due strutture dati globali**
- _tabella dei file attivi_: per ogni file aperto contiene una copia dell' i-node
- _tabella dei file aperti di sistema_: ha un elemento per ogni operazione di apertira relativa a file aperti ( I/O pointer e un puntatore all' i-node )
