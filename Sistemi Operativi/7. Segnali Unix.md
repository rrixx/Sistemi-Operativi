un segnale **notifica un evento** ad un destinatario, e può essere inviato dal kernel a processi utente o da processi ad altri processi utente.

quando un processo riceve un segnale esso può comportarsi in 3 modi 
- **gestire** il segnale con funzione di _handler_
- **eseguire un'azione predefinita da SO** (default)
- **ignorare** il segnale 

nei primi due  casi il processo reagisce in modo asincrono interrompendo la propria esecuzione per eseguire un azione associata per poi tornare all'istruzione successiva del codice interrotto 

in Unix ogni segnale è associato ad un evento e prevede una specifica azione di default 
per riferirsi ai segnali
[[segnali nel sistema]]

N.B. esistono 2 segnali che non possono essere gestiti esplicitamente e sono SIGKILL e SIGSTOP, si esegue dunque sempre l'azione di default 

_**SYSTEM CALL signal**_
ogni processo può gestire esplicitamente un segnale tramite la system call: 

```c
void (* signal(int sig, void (*func)()))(int);
```

- sig= intero che individua il segnale da gestire
- func= puntatore alla funzione handler (SIG_IGN segnale ignorato, SIG_DFL azione default)
- signal() ritorna un puntatore alla funzione precedente gestore del segnale o SIG_ERR in caso di errore 

"puntatore alla funzione signal che prende in ingresso l'int che identifica il segnale e il puntatore alla funzione hanlder"
[[esempio signal]]  

**gestione segnale (hanlder)**

```c

```

_**SYSTEM CALL kill**_
permette ad un processo di mandare un segnale ad un altro processo

```c
int kill(int pid, int sig);
```

- sig l'intero che individua il segnale da gestire
- pid può essere: 
		_pid > 0_: è il pid del solo destinatario
		_pid = 0_: segnale spedito a tutti i processi appartenenti al gruppo 
		_pid <-1_: segnale spedito a tutti i processi appartenenti alla gerarchiua di cui pid è la radice 
		_pid == -1_: vari comportamenti possibili 
[[esempio system call kill]]


_**SYSTEM CALL sleep**_
- provoca la sospensione del processo per N secondi (massimo)
- se il processo dormiente riceve un segnale durante il periodo di sospensione viene risvegliato **prematuramente** 

```c
unsigned int sleep(unsigned int N);
```

ritorna : 
- 0 se la sospensione non è stata interrotta
- il numero di secondi non utilizzati dell'intervallo, se è stato svegliato al tempo Ns    restituisce N-Ns
[[esempio system call sleep]] 


_**SYSTEM CALL alarm_
- imposta un **timer** che dopo N secondi invia al processo il segnale SGNALARM

```c
unsigned int alarm(unsigned int N)
```

ritorna: 
- 0 se non vi erano timeout impostati finora
- il numero di secondi mancante alla fine del timeout precedente 
questa system call non è sospensiva e l'azione di default associata è la terminazione


_**SYSTEM CALL  pause**_
sospende il processo fino alla ricezione di qualunque segnale

```c
int pause(void);
```
ritorna sempre -1

[[esempio system call pause]]
