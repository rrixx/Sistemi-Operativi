un _thread_ è un singolo flusso sequenziale di controllo all'interno di un processo 

in Java sono degli oggetti attivi che eseguono delle sequenze di istruzioni, è possibie crearli dinamicamente attivando concorrentemente le loro esecuzioni

## _**Thread come estensione della classe Thread**_
- _run()_: definisce l'insieme delle istruzioni (da ridefinire per ogni sottoclasse)
- _new()_: crea un nuovo thread
- _start()_: per attivare un thread

```java
class SimpleThread extends Thread {
    public SimpleThread(String str) {
        super(str); //costruttore della superclasse con parametro str
    }

    public void run() { //ridefinizione metodo run 
        for (int i = 0; i < 10; i++) {
            System.out.println(i + " " + getName());
            try {
                sleep((long) (Math.random() * 1000));
            } catch (InterruptedException e) {}
        }
        System.out.println("DONE! " + getName());
    }
}

public class EsempioConDueThreads {
    public static void main(String[] args) {
        SimpleThread st1 = new SimpleThread("Pippo"); 
        st1.start();
    }
}
```

## _**Thread come implementazioni dell'interfaccia Runnable**_ 
- metodo _run()_ ridefinito per ogni implementazione dell interfaccia 
- costruttore _new()_ 
- si passa al costruttore il parametro che è _l'oggetto che implementa Runnable_
- si esegue il thread con _start()_ 

```c
class EsempioRunnable extends MiaClasse implements Runnable {
    
    // Il costruttore qui è implicito (Default Constructor)

    @Override
    public void run() {
        for (int i = 1; i <= 10; i++) {
            System.out.println(i + " " + i * i);
        }
    }
}

public class Esempio {
    public static void main(String args[]) {
        // Creazione dell'oggetto che contiene la logica (il target) implementa Runnable
        EsempioRunnable e = new EsempioRunnable();
        
        // Creazione del contenitore (il Thread) passando il target 
        Thread t = new Thread(e);
        
        // Avvio del thread
        t.start();
    }
}
``` 

## _**SINCRONIZZAZIONE DI THREAD**_
dato che differenti thread occupano la stessa area di memoria (heap)

**Attesa terminazione thread: join()**
per sincronizzarsi con la terminazione di altri thread 
```java
// Esempio di utilizzo del metodo join()
public class EsempioJoin {
    public static void main(String[] args) {
        
        // 1. Creazione del thread figlio
        Mythread T = new Mythread();

        // 2. Avvio del thread figlio
        T.start();

        // ... Qui il thread principale (main) può fare altre operazioni in parallelo ...

        try {
            // 3. Attesa della terminazione
            // Il thread corrente (main) si ferma qui finché T non ha finito
            T.join(); 
        } catch (InterruptedException e) {
            e.printStackTrace();
        }

        // 4. Da qui in poi, siamo sicuri che T è terminato
        System.out.println("Il thread figlio T ha terminato. Proseguo con il main.");
    }
}
```

## _**MUTUA ESCLUSIONE**_
- ad ogni oggetto java associa un oggetto _object lock_ che ne indica lo stato di utilizzo
- utilizzando la keyword _synchronized_ si può operare sulle sezioni critiche di un oggetto, in quanto il compilatore inserisce automaticamente un prologo per l'acquisizione dell'object lock e un epilogo per il suo rilascio

```java
// creazione oggetto lucchetto (Lock)
Object mutexLock = new Object(); 

public void M() {
    // acquisizione del lock
    synchronized (mutexLock) { 
        // SEZIONE CRITICA 
        // solo un thread alla volta può trovarsi qui
        // gli altri in attesa fino al rilascio del lock
        
        < sezione di codice critica >; 
    } 
    // lock rilasciato automaticamente qui.
}
```

con lock occupato entrano nella _coda entry set di ob_ (coda del lock)

```java
//esempio classe contatore 
public class contatore {

	//metodo synchronized
    public synchronized void incrementa() {

    }

    public synchronized void decrementa() {
        i--;
    }
}
```

## _**SEMAFORI IN JAVA**_
presenti nella classe _Semaphore_ (java.util.concurrent.Semaphore) con i segienti metodi:
- _acquire()_: implementa p()
- _release()_: implementa v()
```java
//inizializzazione ad un valore k
Semaphore s=new Semaphore(k);

s.acquire();
s.release();
```
